Data Migration

You always need a method to fetch data, whether it's a one-time task or a recurring process. The ETL/ELT method you choose depends on the situation. For recurring data retrieval, you might use coding languages like Python or C#, or leverage modern tools like Azure Data Factory (ADF) and Databricks, which are popular among the companies we work with. If you're dealing with legacy systems that haven't migrated to the cloud yet, SSIS could be your go-to option. It's challenging to demonstrate our work in those modern tools, but here's an example of a recent project. We had to onboard a client to our application, which required a one-time data import. I took the raw data provided by the client, imported it into a table I created in SQL, and then inserted the data into the corresponding tables. I've edited the code to hide real table names and confidential data. Enjoy the read!

Procedures

When you work with SQL frequently, it's smart to start using procedures for most tasks! On the Data team, we created procedures for the main dataset of every single report. We avoided writing SQL directly in SSRS and preferred using views in PowerBI. We did this for several reasons: primarily for maintenance, to keep all the code centralized within the database rather than embedded in reports is a live saver, and secondly, to enhance the performance of the reports. These are some of the types of procedures we would write or amend on a daily basis.
