Data Migration

You always need a method to fetch data, either once or multiple times. The ETL method you use to get your data depends on the situation. If you need to get it from a source more than once, then you would use SSIS (if you are using SQL Server) or Power Query Editor(if you are using PowerBI); in this case, we had to onboard a client to our application which required a data import that had to be run once. I took the raw data given to us by the client and imported it into a table I created in SQL, from there I inserted the data into the corresponding tables, I edited to code to hide real table names, as well as confidential data, enjoy the read!

Procedures

When you work with SQL a lot, it's a good idea to start using procedures... FOR EVERYTHING! In the BI team, we created procedures for every single report's main dataset, we tried to steer clear of ever writing SQL inside of SSRS. We did this for multiple reasons, firstly to have all the code centralised within SSMS, secondly, to improve performance on the report itself. These are some of the types of procedures we would write/amend on a daily basis.
